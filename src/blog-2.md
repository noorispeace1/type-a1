Mastering DRY Code with TypeScript Utility Types: Pick and Omit
One of the core principles of software engineering is DRY (Don't Repeat Yourself). In TypeScript, we often find ourselves redefining similar interfaces for different parts of an application. This is where Pick and Omit become essential tools.

The Problem: Interface Duplication
Imagine a "Master Interface" for a User in a database. When creating a new user or displaying a profile, you might not need every single field.

TypeScript

interface User {
    id: string;
    name: string;
    email: string;
    createdAt: Date;
    isAdmin: boolean;
}
Instead of creating a new ProfileDisplay interface from scratch, which leads to maintenance headaches if the User interface changes, we use Utility Types.

Creating Specialized "Slices"
1. The Pick Utility
Pick allows you to create a new type by choosing specific keys from an existing interface.

TypeScript

// We only need name and email for the public profile
type PublicProfile = Pick<User, "name" | "email">;

const displayUser: PublicProfile = {
    name: "John Doe",
    email: "john@example.com"
};
2. The Omit Utility
Omit does the opposite: it creates a type by picking all properties except the ones you specify.

TypeScript

// When creating a user, the ID and Date are usually generated by the DB
type UserCreationRequest = Omit<User, "id" | "createdAt">;

const newUser: UserCreationRequest = {
    name: "Alice",
    email: "alice@dev.com",
    isAdmin: false
};
Why This Matters
By "slicing" your master interface, you ensure a single source of truth. If you rename name to fullName in the master User interface, TypeScript will automatically flag errors in your Pick and Omit types, forcing you to update your code consistently.
